# BigUp QA - Automated Testing Suite

This repository contains the automated testing suite for BigUp, focused on verifying core features like Predictions, Biggie AI Chat, Global Chat, Quests, and Stream Replay using Playwright.

## 📁 Directory Structure

- `tests/`: Main directory for test specifications.
  - `pages/`: Page Object Models (POM) for encapsulated UI logic.
  - `auth.setup.ts`: Handling of global authentication state.
- `.github/workflows/`: CI/CD pipelines split by feature area for efficient execution.
- `playwright.config.ts`: Centralized configuration for browsers, timeouts, and environment settings.

## 🚀 Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (LTS recommended)
- [npm](https://www.npmjs.com/)

### Installation
1. Install project dependencies:
   ```bash
   npm install
   ```
2. Install Playwright browsers:
   ```bash
   npx playwright install --with-deps
   ```

### Authentication Setup
Local tests require a valid authentication state.
1. Ensure you have a `config.json` file in the root directory.
2. The file should contain a valid login session in JSON format (usually generated by `auth.setup.ts` or provided via secrets in CI).

## 🧪 Running Tests

Run all tests:
```bash
npx playwright test
```

Run a specific test suite:
```bash
npx playwright test tests/biggie.spec.ts
```

Run tests in headed mode (to see the browser):
```bash
npx playwright test --headed
```

View the last test report:
```bash
npx playwright show-report
```

## 📋 Test Case Catalog

### 🤖 Biggie AI (`biggie.spec.ts`)
- **TC-AI-001: Direct Inquiry**: Verifies the AI responds correctly to natural language prompts.
- **TC-AI-003: History Persistence**: Ensures chat history is saved and accessible across navigation.
- **TC-AI-004: Send Button Check**: Verifies the visual send button behavior (Regression check for a known UI issue).

### 💬 Global Chat (`chat.spec.ts`)
- **TC-CHAT-001: Message Delivery**: Standard message sending with emoji and timestamp verification.
- **TC-CHAT-002: Empty Message Validation**: Ensures empty messages cannot be sent.
- **TC-CHAT-003: @Biggie Mention**: Verifies triggering the AI via mentions in the global chat and receiving a response in a thread.

### 📈 Predictions (`predictions.spec.ts`)
- **TC-PRED-001: Homepage Feed Quick Predict**: Direct entry from the main feed cards.
- **TC-PRED-ACT-001: Flex/Power Play Toggle**: Configures entries with different strategies in the Activity Page.
- **TC-PRED-ACT-002: Persistence Check**: Verifies selection state remains intact when opening the entry drawer.

### 🏆 Quests Hub (`quests.spec.ts`)
- **TC-QUEST-001: Infrastructure Check**: Verifies the Quests Hub loads correctly with current progress indicators.
- **TC-QUEST-002: Dynamic Progress Update**: Verifies that performing a prediction correctly increments related quest progress in real-time.

### 🎥 Stream Replay (`stream.spec.ts`)
- **TC-STREAM-001/002: Playback Controls**: Verifies video loading, play, and pause functionality.
- **TC-STREAM-003/004: Timeline Manipulation**: Verifies that seeking and skipping (forward/backward) work accurately.
- **TC-STREAM-005: Context Switching**: Ensures video playback continues uninterrupted when switching between chat and game tabs.

## ⚙️ CI/CD Integration

Tests are automatically executed via GitHub Actions on every push to `main` and manually via `workflow_dispatch`. Each feature has a dedicated workflow in `.github/workflows/` to allow for targeted testing and faster feedback loops.
